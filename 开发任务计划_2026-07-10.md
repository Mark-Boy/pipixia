## 🔜 近期计划（Sprint 3-4：质量提升 + 技术债清理）

### Sprint 3 — 质量提升（预计 9h）

#### S07 — 图片合成（P2，4h）⭐ 下一步
**目标**：图片翻译后能合成泰语文字覆盖

**验收标准**：
- [ ] OCR 提取图片文字（PaddleOCR / Tesseract）— PaddleOCR 已集成
- [ ] 翻译文字为泰语
- [ ] OpenCV/Pillow 合成新图片
- [ ] 支持多种字体大小和颜色
- [ ] Worker 任务 `process_image` 调用链路完整

**涉及文件**：
- `worker/tasks.py` — 新增 `process_image` 任务
- `api/services/translator.py` — 泰语翻译能力
- `api/models/product.py` — 图片字段

**依赖**：无（PaddleOCR 已安装）

---

#### S08 — 路由/Schema 层测试（P2，3h）
**目标**：测试覆盖率提升到 80%+

**验收标准**：
- [ ] 每个路由端点有 happy path + error path 测试
- [ ] Schema 验证测试（边界条件）
- [ ] Worker 任务单元测试
- [ ] 前端 API 服务集成测试
- [ ] `pytest` 全部通过

**涉及文件**：
- `tests/` — 新增路由层测试
- `api/routers/` — 被测试路由

---

#### S09 — SEO 标签泰语分词（P2，2h）
**目标**：商品标题/描述自动生成 SEO 标签

**验收标准**：
- [ ] 泰语分词（PyThaiNltk / custom tokenizer）
- [ ] 关键词提取（TF-IDF 简化版）
- [ ] 标签去重 + 长度限制
- [ ] 翻译后标签保留核心含义

**涉及文件**：
- `api/services/translator.py`
- `worker/tasks.py`

---

### Sprint 4 — 技术债清理（预计 5h）

#### S10 — API 响应格式统一（P3，1h）
**目标**：所有 API 响应遵循统一格式

**验收标准**：
- [ ] 成功响应: `{"code": 200, "data": {...}, "message": "success"}`
- [ ] 错误响应: `{"code": 400, "data": null, "message": "错误描述"}`
- [ ] 分页响应: `{"code": 200, "data": {...}, "meta": {"page": 1, "total": 100}}`
- [ ] FastAPI `exception_handler` 统一捕获

**涉及文件**：
- `api/main.py` — exception handler
- 所有 `routers/*.py`

---

#### S11 — Dockerfile 编写与测试（P3，2h）
**目标**：三个服务都能独立 Docker 构建运行

**验收标准**：
- [ ] `api/Dockerfile` 多阶段构建，镜像 < 200MB
- [ ] `worker/Dockerfile` 多阶段构建
- [ ] `dashboard/Dockerfile` 生产构建验证
- [ ] `docker compose build` 成功
- [ ] `docker compose up` 全服务启动成功

**涉及文件**：
- `api/Dockerfile`
- `worker/Dockerfile`
- `dashboard/Dockerfile`
- `docker-compose.yml`

---

#### S12 — 配置管理增强（P3，2h）
**目标**：从 YAML 配置文件迁移到 DB 配置表

**验收标准**：
- [ ] `site_config` 模型支持动态配置
- [ ] API 端点支持配置 CRUD
- [ ] 配置变更实时生效（无需重启）
- [ ] 敏感字段加密存储

**涉及文件**：
- `api/models/site_config.py`
- `api/config.py`
- `api/routers/settings.py`
