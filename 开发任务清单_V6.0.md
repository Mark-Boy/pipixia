## Sprint 1 — 核心功能补齐（阻塞性任务）

### S01 — 利润熔断 Celery 任务完整实现（P0，3h）

**目标**：`worker/tasks.py` 中 `profit_circuit_breaker` 任务从占位符变为真实逻辑

**验收标准**：
- [ ] 查询所有 `audited` 状态的商品
- [ ] 使用实时汇率（从 API/缓存获取）计算实际利润率
- [ ] 利润率低于阈值的商品自动标记为 `blocked` + `risk_status=block`
- [ ] 写入 `risk_logs` 表记录风控日志
- [ ] 发送 Telegram 告警（如果配置了 bot token）
- [ ] 返回统计结果（下架数量、检查数量）

**涉及文件**：
- `worker/tasks.py` — `profit_circuit_breaker` 任务
- `api/services/exchange.py` — 汇率服务
- `api/models/risk_log.py` — 风控日志模型
- `api/config.py` — 配置

### S02 — 订单 Webhook 完整处理链路（P0，4h）

**目标**：`webhooks.py` + `worker/tasks.py::webhook_handler` 完成订单状态变更全链路

**验收标准**：
- [ ] 订单创建 → 创建本地 listing 记录
- [ ] 订单发货 → 更新库存数量
- [ ] 订单取消 → 释放库存（回滚）
- [ ] 订单完成 → 触发利润核算
- [ ] 评价事件 → 负面评价触发补偿策略（已在 tasks.py 中）
- [ ] 库存变化事件 → 低库存告警

**涉及文件**：
- `api/routers/webhooks.py`
- `worker/tasks.py` — `webhook_handler`
- `api/models/listing.py` — listing 模型
- `api/models/shop.py` — shop 模型

### S03 — Celery Beat 验证与修复（P0，2h）

**目标**：验证 Celery Beat 调度器正常运行，修复问题

**验收标准**：
- [ ] `celery beat` 进程正常启动
- [ ] 定时任务按 schedule 触发（日志可见）
- [ ] `task_always_eager=False` 时 Redis broker 正常工作
- [ ] Beat 和 Worker 日志分离可查

**涉及文件**：
- `worker/celery_app.py` — Beat schedule
- `docker-compose.yml` — Worker 服务配置
- `Makefile` — 启动命令
