## ✅ T001 — 用户认证模块（JWT Login/Register）

### 1. 需求实现检查清单

| # | 需求项 | 状态 | 实现位置 |
|---|--------|------|----------|
| 1.1 | 用户注册（用户名+密码+角色） | ✅ | `api/routers/auth.py` `POST /register` |
| 1.2 | 用户登录（用户名+密码） | ✅ | `api/routers/auth.py` `POST /login` |
| 1.3 | Access Token（2h 有效期） | ✅ | `api/services/auth.py` `create_access_token()` |
| 1.4 | Refresh Token（7d 有效期） | ✅ | `api/services/auth.py` `create_refresh_token()` |
| 1.5 | Token 刷新（Rotation 轮换） | ✅ | `api/routers/auth.py` `POST /refresh` |
| 1.6 | Token 黑名单（登出失效） | ✅ | `api/services/auth.py` `blacklist_token()` |
| 1.7 | Cookie 安全属性（HttpOnly + Secure + SameSite） | ✅ | `api/routers/auth.py` `set_cookie()` |
| 1.8 | JWT 鉴权依赖注入 | ✅ | `api/services/auth.py` `get_current_user()` |
| 1.9 | 密码哈希（bcrypt） | ✅ | `api/services/auth.py` `get_password_hash()` + `verify_password()` |
| 1.10 | 用户信息查看（/me） | ✅ | `api/routers/auth.py` `GET /me` |

### 2. API 端点清单

| 方法 | 路径 | 功能 | 鉴权 |
|------|------|------|------|
| POST | `/api/v1/register` | 用户注册 | 无 |
| POST | `/api/v1/login` | 用户登录 | 无 |
| POST | `/api/v1/refresh` | Token 刷新 | Cookie (refresh_token) |
| POST | `/api/v1/logout` | 用户登出 | Access Token |
| GET | `/api/v1/me` | 获取当前用户信息 | Access Token |

### 3. 核心函数清单

| 函数 | 文件 | 功能 |
|------|------|------|
| `create_access_token(user_id, role, expires_delta)` | `api/services/auth.py` | 创建 Access Token |
| `create_refresh_token(user_id)` | `api/services/auth.py` | 创建 Refresh Token |
| `decode_token(token)` | `api/services/auth.py` | 解码并验证 Token |
| `refresh_tokens(refresh_token, db)` | `api/services/auth.py` | Token 轮换 |
| `blacklist_token(token, db)` | `api/services/auth.py` | Token 加入黑名单 |
| `is_token_blacklisted(token, db)` | `api/services/auth.py` | 检查 Token 是否黑名单 |
| `get_current_user()` | `api/services/auth.py` | JWT 鉴权依赖注入 |
| `get_password_hash(password)` | `api/services/auth.py` | bcrypt 哈希密码 |
| `verify_password(plain, hashed)` | `api/services/auth.py` | 验证密码 |

### 4. 安全特性验证

- ✅ JWT 使用 HS256 签名（`settings.ALGORITHM = "HS256"`）
- ✅ Access Token 独立于 Refresh Token，过期时间短（2h）
- ✅ Refresh Token 轮换：旧 Refresh Token 刷新后立即加入黑名单
- ✅ Cookie 设置：`HttpOnly=True`（防 XSS）、`Secure=True`、`SameSite="lax"`
- ✅ 密码使用 bcrypt 哈希（自动加盐）
- ✅ 密码复杂度校验（至少 8 位）

### 5. 验收结论：**✅ 通过**
